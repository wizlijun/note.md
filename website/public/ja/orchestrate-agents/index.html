ひとつの Vault、多くのエージェント。指揮するのはあなた。

誰も警告してくれないロックインは、アプリではない — エージェントだ。知識をプレーンファイルに置けば、どの単一の AI もそれを所有できない。Cowork が下書きし、Codex がリファクタリングし、ChatGPT が生成し、ローカルエージェントが秘密を守る — そしてペンを握るのはあなただ。

Vault は中立地帯だ

ほとんどの AI ツールは、あなたの思考の住処になりたがる。知識は彼らのデータベース、注釈は彼らの形式、エージェントは彼らが出荷するもの、モデルは彼らがあなたを縛りつけるもの。すると「どの AI を使うか？」は「全部を移行するか？」になり — あなたは一社のロードマップの中に囲い込まれる。

note.md の Vault はそれをひっくり返す。Vault は共有された規約を持つプレーンな markdown のフォルダだ — ハウスルールを述べる AGENTS.md、正確な参照のための ((file#b-xxxxxx)) ブロック引用、あなたの判断を保持するサイドカー .note.md ファイル、そしてひとつの名前空間のための [[wikilinks]]。これらの規約は公開プロトコルだ。どのエージェントも読める、アダプタは要らない。エージェントとモデルは交換可能な働き手になり、変わらない唯一のものが Vault だ。それは git リポジトリで、全員がそこにコミットしている。

各仕事を、いちばん得意な相手に割り当てる

すべてに最も強い単一のエージェントはいない。だからひとつに全部をやらせるな — ラインを組み、各ツールを最も強いステーションに置け：

仕事ごとにエージェントとモデルを選ぶ — 仕分けには安くて速いモデル、推論にはフロンティアモデル、プライベートなものにはローカルモデル。Vault はどれでも構わない。ただ彼らが受け渡すファイルを保持するだけだ。

実際のループ

これが本物のパイプラインだ。ひとつの Vault の上で、四つのツールと三つのモデル：

  1. OpenClaw が夜通し走り、生のノートの束を drafts/*.md に処理する。
  2. あなたはその下書きを、慎重なモデルの Claude Cowork に渡す — 「これをレビューして改稿し、怪しいものはフラグを立てて」。
  3. ChatGPT がヒーロー画像を一括生成し、各文書の _files/ フォルダに入れる。
  4. 仕上がった文書は note.md に着地し、そこであなたが読み、言い過ぎを切り、大事なものをハイライトし、あなたにしか書けないメモを残す。

四つのツール、三つのモデル、ひとつの Vault、ひとりの指揮者。誰も記憶を共有したり、私的なプロトコルを話したりする必要はなかった — 彼らはディスク上の .md ファイルを受け渡し、あなたのサイドカー .note.md の注釈が、次の相手への操舵信号になった。

なぜファイルがそれを成立させるのか

  • 反ロックイン、もう一層深く。ファイル・オーバー・アプリはアプリから解放する。これはエージェントとモデルから解放する。今日の最良のモデルは来月には置き換わる — あなたの知識がそれと一緒に動く必要はない。
  • プラットフォームなしのコラボレーション。エージェントはディスク上で .md を受け渡す — 共有メモリなし、私的な API なし、プラグインストアなし。あるエージェントの出力が、次のエージェントの入力になる。
  • あなたはループの中、チェックポイントに留まる。端から端まで走るブラックボックスではない。品質管理に人間を置いた、ステーションの並ぶラインだ。エージェントが書き、レビューし、図解する — 何を世に出すかを決めるのはあなただ。
  • それでもただのファイル。オーケストレーションのデータベースなし、隠れた状態なし。ルールは AGENTS.md、出力は .md、判断は .note.md — すべて Obsidian でも、CLI でも、どのエージェントでも読める。note.md を差し替えても、Vault は依然として全員の共有ワークスペースだ。

Vault を git に入れておけば、すべてのエージェントの書き込みは diff 可能、帰属可能、巻き戻し可能になる — エージェントの厄日は git revert 一発で済み、悲劇にはならない。

セットアップ

  1. Vault のルートに AGENTS.md を置く — llms-full.txt から規約を取り、ハウスルールを足す（鉄則：エージェントは決して *.note.md サイドカーに書き込まない）。
  2. 各エージェントを同じフォルダに繋ぐ：OpenClaw、Cowork、Codex、ChatGPT、Hermes。
  3. 結果を note.md で読み、注釈する。次のエージェントには先にサイドカーを読ませる。ループはあなたのディスクの上で閉じる。

FAQ

異なる AI エージェントが本当にひとつの Vault を共有できるか？

できる — それが設計だ。Vault はプレーンな markdown に、規約を述べたひとつの AGENTS.md。Claude Cowork、Claude Code、Codex、ChatGPT、OpenClaw、Hermes はすべてそのファイルを読み書きするので、各タスクを最適なエージェント（とモデル）に振り分けられる。Vault を git に入れておけば、すべての書き込みは diff 可能で巻き戻せる。

エージェントたちはどうやって互いに仕事を受け渡すのか？

ファイルを通じてだ。あるエージェントが markdown を Vault に書き、次のエージェントがそれを入力として読む。あなたの注釈はサイドカーの .note.md ファイルに住み、操舵信号として働く — エージェントは次のパスの前にあなたの余白を読む。共有メモリも私的なプロトコルも要らない。

これには専用のオーケストレーションツールや MCP サーバーが要るか？

要らない。オーケストレーションはあなた自身で、媒体はファイルシステムだ。中央データベースも隠れた状態もない — ルールは AGENTS.md、出力は .md、判断は .note.md。ツールインターフェースを好むエージェントのために Vault MCP サーバーがロードマップ上にあるが、プレーンなファイルはもう今日から動く。

なぜ全部にひとつの AI を使わないのか？

単一のエージェントがすべてに最も強いわけではないからだ。夜通しの自動化、慎重なレビュー、速い画像生成、プライベートなローカル作業、そして最終判断は、それぞれ最適なツールの異なる別々の仕事だ。それらを — 自分が所有するファイルの上で — 専門家たちに分けるほうが、ひとりの万能選手に全部やらせるより勝る。そして、どの働き手も差し替えられる自由を保てる。
