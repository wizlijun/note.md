note.md + Claude Cowork

Cowork はクラウドで Claude を走らせ、Mac のフォルダに接続する。Vault を接続すれば、Claude の生産物はすべて、読めて、マークできて、手元に残るものになる。

なぜこの組み合わせが効くのか

Cowork の納品物は圧倒的に markdown だ。調査レポート、計画、仕様、草稿。デフォルトではそれらは散らかる — ダウンロードがここに、会話の添付があそこに。代わりに Cowork を note.md の Vault に向ければ、出力はリーディングループの住む場所に着地する。すべてのレポートに住所ができ、すべての読了が判断のサイドカーを残し、次の Cowork セッションには先にそのサイドカーを読ませられる。

セットアップ

  1. Claude デスクトップアプリで、Vault のフォルダを Cowork セッションに接続する（「フォルダを追加」）。
  2. Vault のルートに AGENTS.md を置く（規約の要約：llms-full.txt）— Claude は自動で読み、ハウスルールに従う。
  3. 納品物を Vault に保存するよう Claude に頼む。例：research/2026-07-11-competitor-scan.md。
  4. note.md で読む。ハイライトとメモはサイドカーの .note.md に保存される。
  5. 次のセッションで一行：「先週書いたレポートの .note.md サイドカーを読んで、私の余白に応えて」。ループが閉じる。

コツ

  • Claude に [[wikilinks]] と [[yyyy-MM-dd]] の日付形式を使わせる。文書が Vault のリンクグラフの外を漂わず、その一部になる。
  • Vault は git に入れておく — Cowork の書き込みは diff で追えるし、Cowork のファイルバージョニングとあなたのそれが喧嘩することもない。

FAQ

Claude は Vault の規約を尊重するか？

する。フォルダのルートの AGENTS.md に書いておけばいい — Claude Code と Cowork がエージェント指示ファイルを読むのは標準的な作法だ。

Claude は私の注釈を読めるか？

それこそが本題だ。サイドカーの .note.md はプレーンな markdown。どのセッションにでも読ませれば、あなたが何をハイライトし、何を疑問視したかを正確に見る。
