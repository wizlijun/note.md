# 周检视（Weekly Review）

把 vault 近 7 天的日记与埋点数据整理成一份周检视。核心边界：**agent 只做数据预填与证据整理，检视判断留给人**。检视自动化系统的动作必须发生在系统外——因此 B 区答案与 C 区决定禁止代笔；代笔会把检视变成又一份可再生成的摘要，恰好违背它要防的事。

## 参数（只问这两项，能推断就不问）

- `vault_dir`：vault 根目录。用户已挂载文件夹时直接用它。
- `focus`：收敛层方向列表（如 aicoding、bushcraft）。上一期周检视或记忆中已有时直接沿用。

其余一切内化为下述默认值，仅当用户明确要求时才覆盖。

## 内化默认值

**七滤镜** = focus（收敛层，主动进攻）+ 生活层五滤镜（守护型）：儿子教育、家庭关系、金融理财、健康、朋友与客户关系。"滤镜外立项"按全部七个判定。

**数据源（相对 vault_dir）**

| 数据 | 位置 |
| --- | --- |
| 日记 | `diary/YYYY-MM-DD-diary-*.md`，取近 7 天有日记的日子 |
| 阅读统计 | `stat/YYYY-MM-DD-daily-stat.md`（首段有总时长与最高注意力文档，表格有逐文档时长/标注数） |
| 结论登记簿 | `decision/open.decision.note.md` + `decision/_scoreboard.jsonl` |
| 话题 md | vault 根目录 `YYYY-MM-DD-来源-slug.md`；伴生 `.note.md` 是深读证据 |
| 生成物库存 | `youtube-summaries/`、`youtube-daily/` |
| 输出 | `weekly-review/YYYY-Www-weekly-review.md` |
| 上周基线 | 上一期 weekly-review 的 A 表；找不到才从上周日记与统计估算，数值前标 ≈ |

某个路径不存在时，先 `ls` vault 根目录找同义目录（如 dairy/），仍找不到再问用户。

**口径**

- 立项 = 窗口内新建话题 md；同一话题多个文件合并计 1
- 深读完成 = 有阅读时长证据，且有标注或 `.note.md`
- WIP = 已立项、未深读、也未显式关闭（没写过"为什么不读了"）
- 共鸣/不清晰 = 立项触发信号；vault 暂无触发信号记录时按日记证据事后归类，并在 A 表标"agent 事后归类，待核"
- 阅读时长 = daily-stat 合计；单文档单日 >3h 疑似挂机，按 1h 折算并注明原始值
- 观点改变分两个口径分开报：登记簿口径（被修正/推翻的旧结论）、日记口径（"改变的想法"中的实质翻案）
- 元工作 = 关于系统本身的产出（diary 管线、时间块、决策管线、session 导出、周检视等），按时长估占比；本检视文档也计入

**警戒线**（超线在 A 表该行加 ⚠️ 与实际值）：立项:深读 > 2:1 收紧；WIP > 5 必须先淘汰；共鸣占比 > 70% 警报；滤镜外连续 4 周 = 0 警报；未读生成物 >30 天到期降级；原文:摘要阅读比持续走低 = 认知债累积；结论复核到期未复核列入 B3；观点改变连续 2 个月 = 0 红色警报；元工作 > 20% 警报。

## 流程

1. **定窗口**：近 7 天有日记的日子（通常截至昨天）。标题用 ISO 周编号；窗口跨周时在首行引言注明。
2. **读数据**：窗口内日记全文；每日 daily-stat 的总时长与高注意力条目；结论登记簿全文；根目录窗口内新建的话题 md 及 `.note.md`；生成物新增计数。
3. **算指标**：九项指标 + 上周基线对比。
4. **产出文件**（模板见下）：
   - A 表全填。
   - B 区每问预填三行——**事实**（数字与警报）、**口径**（这题的数字怎么算的）、**证据**（嵌套大纲，条目优先级从上至下：淘汰候选按"最无阅读痕迹"、深读按时长、复核按紧迫度、元工作按耗时；不写长句串列）——然后 `> 答：` 留白。
   - C 区各项留空。
   - D 一句话可代拟：现代白话鲁迅体，只记事、不抒情、克制短句（例："本周立项十五，深读七，一个未扔。"）。
5. **交付**：给出文件，并单独提醒最近的结论复核到期项。

## 采集缺口处理

某项无数据（如触发信号未记录、未读生成物无埋点）时，不编数字：如实写"无埋点"并注明缺口依赖的待办。缺口本身就是检视信息。

```markdown

## A. 数据面板（agent 预填，人只核对）

| 指标 | 本周 | 上周 | 警戒线 |
| --- | --- | --- | --- |
| 立项数 / 深读完成数 | n / n | n / n | 比例 > 2:1 收紧 |
| 在读队列（WIP） | n | n | > 5 必须先淘汰 |
| 共鸣 : 不清晰（立项触发比） | n : n | n : n | 共鸣占比 > 70% 警报 |
| 滤镜外立项数 | n | n | 连续 4 周 = 0 警报 |
| 未读生成物（>30 天） | n | n | 到期自动降级 |
| 原文阅读时长 : 摘要阅读时长 | n : n | n : n | 持续走低 = 认知债累积 |
| 结论登记簿：新增 / 复核到期 | n / n | n / n | 到期未复核即列入 B3 |
| 本月观点改变数（累计） | n | — | 连续 2 个月 = 0 红色警报 |
| 元工作占比（关于系统本身的产出） | n% | n% | > 20% 警报 |

## B. 五问（人工手写，每问一两行）

### 1. 滤镜问：七个滤镜还对吗？有该退役或补位的方向吗？

- **事实**：（立项的滤镜分布；零触发滤镜）
- **口径**：（七滤镜构成；滤镜外判定）
- **证据**（按占比从高到低）
  - （focus 方向逐条，条目多时按主题分组）
  - （滤镜外条目）

> 答：

### 2. 队列问：该淘汰哪个话题？为什么不读了（一行）？

- **事实**：（立项/深读比及警报；WIP 数；生成物库存；阅读时长与认知债信号）
- **口径**：（立项、深读、WIP、挂机折算）
- **证据**
  - WIP（淘汰候选，越靠上越无阅读痕迹）
  - 深读完成（时间投入从高到低，附日期与时长/标注）

> 答：

### 3. 结论问：本周新增结论是什么？有没有该翻案的旧结论？

- **事实**：（登记簿新增/复核到期；两口径观点改变数）
- **口径**：（登记簿与日记口径的区分）
- **证据**
  - 簿内新增（按复核紧迫度，附信心与复核日）
  - 日记翻案（未入簿的，附日期）

> 答：

### 4. memory 问：本周摘要有迎合迹象吗？memory 有该纠偏的固化偏好吗？

- **事实**：（共鸣占比及警报；反迎合修补落地状态）
- **口径**：（触发信号归类方式与可靠性）
- **证据**
  - 不清晰立项逐条（附缺口类型）
  - 共鸣立项概括
  - 未落地修补（附出处）

> 答：

### 5. 行为问：本周哪条结论真的改变了行为或决策？

- **事实**：（最清晰的"读→行为"闭环；元工作占比及警报）
- **口径**：（行为验收标准；元工作范围）
- **证据**
  - 行为闭环候选（按强度从高到低）
  - 元工作明细（时长从高到低）

> 答：

- 翻案 / 复核：
```
