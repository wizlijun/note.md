# 决策日志 v1.1 需求

> 依据:`docs/2026-07-24-decision-log-design-review.md`(研究结论)+ 用户裁决(2026-07-24):信心五档、交互用五颗星、顶部一句话价值主张。
> 对照 spec `docs/2026-07-21-decision-log-spec.md`;冲突处以本文为准,采纳后回写 spec。

## R1 顶部价值主张

窗口最顶部(标题栏下)常驻一句话,建立信心:

> **把决定变成可检验的下注:时间会告诉你,你的判断哪里可信、哪里高估。**

- en: *Turn decisions into testable bets — time will show where your judgment can be trusted, and where it runs hot.*
- 静态文案,不可关闭,弱化样式(小字、低对比),四语种。

## R2 信心 = 10 格概率条(取代早先的五星方案)

> 五星对 >50% 的概率有误导:1★ 视觉上像"很低",实际是 55%(勉强过半)。改用 10 格概率计更贴合"下注多确定"的含义。

替换三档按钮。**10 个格子**排成一条:
- **前 5 格**默认闭合、不可点(0–50% 基线——你签字的决策,前提就是你认为"多半会发生",这一半白送)。
- **后 5 格**像进度条一样可选,每格 = 一个信心档,锚定一个概率:

| 选中格 | 锚定 | 标签(zh) |
|---|---|---|
| 第 6 格 | 55% | 勉强过半 |
| 第 7 格 | 65% | 六成把握 |
| 第 8 格 | 75% | 挺有把握 |
| 第 9 格 | 85% | 很有把握 |
| 第 10 格 | 95% | 几乎确定 |

- **鼠标悬停某个可选格**:实时预览填充,并在下方 caption 显示"标签 · ≈NN%"(锚定语义,教育概率思维);同时 aria-label 供无障碍/触屏。
- 前后两半之间留视觉间隙标出 50% 分界。
- **存储改数值**:front-matter / jsonl 里 `confidence: 0.85`(number)。旧枚举读入映射:low→0.6、medium→0.75、high→0.9;档位取最近锚点(`round((p−0.55)/0.1)+1` 截断 1..5,+1e-9 防浮点 tie 翻转)。写盘一律数值。
- 候选 JSON 的 `confidence` 接受 number(0–1)或旧三档字符串。
- 未决卡片显示紧凑只读概率条;裁决锁定区显示只读概率条 + caption;记分牌分桶标签用"≈NN% 标签"。签字后锁定规则不变。
- 组件:`ConfidenceBar.svelte`(readonly / compact 两开关);内部复用 model 的 5 档 `starOf/anchorOf`("star" 命名为历史内部术语,渲染已是格子)。

## R3 记分牌:校准 + 决策分

右栏改为:
1. **校准分桶**(按 5 星分桶):每桶 `命中/总数` + 进度条,桶标签=星+锚定百分比。
2. **决策分**(新,净正和 proper score):每次裁决得
   `score = max(0, round(10 + 40·log₂(2p)))`,p=押中侧的概率(hit→conf,miss→1−conf,partial→固定 p=0.5 即 10 分参与分)。累计总分大数字展示。裁决即得分(参与激励),失准少得但不为负。
3. **样本进度**(保留)。
4. **回避主题**(保留)。
- score 在 verdict 事件 append 时算好冻结入 jsonl;读取时缺失则按公式补算(向后兼容)。
- 措辞永远学习框架:无对错率、无红色警示。

## R4 跳过语义(替换"三振"的计数口径)

到期检查里"本次跳过"展开为三个原因(必选其一才算跳过):

| 原因 | 动作 | strike |
|---|---|---|
| 还没到时候 | check-date 自动 +14 天 | 不计 |
| 已不相关 | 直接 drop 归档(无愧疚文案) | 不计 |
| 先不想面对 | 计 1 次;满 3 次降级归档(现有逻辑) | 计 |

- 整场到期检查没做 → 任何决策不计 strike(现状已如此,保持)。
- 每次跳过记 jsonl 事件 `{ event:"skip", id, reason }`。
- 到期检查结束页:若归档中有降级项,温和提示一行"沉底 N 条,归档列随时捞回"。

## R5 裁决第三问(可选)

第②问答"不会"时,追加单选(可跳过):"哪一环最弱?"
六选一:想错了问题 frame / 没想别的选项 alternatives / 信息不够 information / 没想清楚要什么 values / 推理有漏洞 reasoning / 没真执行 commitment。
落归档 `weakest-element` + verdict 事件 `weakest_element`;记分牌暂不展示(数据先积累)。

## R6 签字模态新增两个可选字段

- **失败预想 premortem**(textarea,🔒签字后锁定),提示语逐字:"假设现在是检查日,这个决策**已经失败了**。最可能的原因是什么?"
- **落选备选项 alternatives**(每行一条,🔒)。
- 候选 JSON 可带 `premortem_hint` / `alternatives` 预填;归档时随条目留存。

## R7 防锚定

裁决模态**不再**用 closure.suggested_outcome 自动预选结果按钮;只有用户在建议条上明确点了"关闭:命中"等动作才预填。证据引文仍置顶展示。

## R8 兼容与迁移

- 读:旧枚举 confidence、无 score 的旧 verdict 事件、无新字段的旧文件全部可读。
- 写:一律新格式。无需一次性迁移脚本(读时映射)。
- agent 契约(`docs/decision-log-agent-prompt.md` + skill):confidence 允许 number;新增可选 `premortem_hint`/`alternatives`;向后兼容。

## 验收

签字:五星可点、星下出现"标签 · ≈NN%"、premortem/备选可留空。裁决:无 AI 预选、答"不会"出现第三问。到期检查:跳过出三选一,"还没到时候"改期不计振。记分牌:顶部价值一句话可见、决策分随裁决增长、校准按星分桶。旧 vault 数据打开不报错。
